# 版本变更记录

> 本记录只保留功能与行为变更。涉及车机底层接口的逆向细节不在此公开。

## v5.11 (versionCode 34) — 2026-06-25
- **修「发命令机器人偶尔没反应」（借鉴同款设备成熟实现）**：对照另一款同型机器人 App 的 BLE 收发，定位到本项目两处可靠性短板并修正——
  - **改用带应答写**：原先优先「无应答写」，丢帧后系统仍回成功、App 无感，命令静默丢失。现优先「带应答写」（特征支持时），ATT 层确认送达、回调状态真实，配合既有失败重试才真正不丢命令；仅在特征不支持时退回无应答。
  - **断链暂存不丢弃**：原先端点未就绪（锁车/睡眠唤醒的重连窗口）时直接丢弃指令。现改为先入队缓存、连上即补发；并加队列封顶（32）与时效淘汰（3s）——超时未送达才丢，避免长断连后唤醒一次性回放大量过期瞬时动作。
  - 中断的在途写与积压指令在重连后自动补发；写特征发现时记录 properties 便于实车确认带应答支持。
- 说明：未改连接参数/写节奏（延迟相关保持原样）。

## v5.10 (versionCode 33) — 2026-06-23
- **摇头晃脑入动作库**：0x25「摇头晃脑」（运动板内置摆头动作，非眼屏表情）接入场景助手动作列表，自定义规则可直接选；控制台「快捷指令」也加了一键按钮。
- **触发源枚举显示订正**：自定义规则的「挡位/座椅就座/胎压状态/驾驶模式」由原来填裸数值改为下拉选项（P/R/N/D、有人/无人、各轮告警、经济/舒适/运动…），用真实车机常量值映射，不再让用户猜数字。
- **预置场景可点击进入详情**：列表里的预置项现在可点开——「急加速急刹」「冷热反应」给出可现场调整的阈值输入框（保存即生效、无需重连），其余预置展示触发→动作说明。
- **修加速度联动「无反应」（根因）**：纵向加速度是传感器原生量，实测静止仅 0.0x 量级（疑似 g 而非 m/s²），原阈值按 m/s² 设的 2.5/3.0 永远够不到。本版默认阈值改为约 g 量级（急加速 0.15 / 急刹 0.20），并可在预置详情里实车标定；监听同时把原始读数落 W 级日志（变化 ≥0.02 才记）便于抓 log 校准。

## v5.9 (versionCode 32) — 2026-06-23
- **修主页只剩一个设备**：v5.8 误把「氛围灯 · ET-ROBOT-01」设备行整块隐藏，导致「设备连接」卡里只显示机器人本体一台、无法分别绑定氛围灯。本版恢复该行，主页重新显示两台设备各自的连接状态与「绑定此设备」按钮。
- **表情字节订正（实车）**：0x26 由「吃零食」改为「吃月饼」；0x25 实测是「摇头晃脑」动作（非眼屏表情），从表情探索器的默认/示例里移除，默认探索字节改 0x14，并在提示里标注 0x25/0x26 的真实含义。

## v5.8 (versionCode 31) — 2026-06-23
- **眼屏表情直控（逆向小程序得到）**：确认表情由运动板 FE55 协议直接驱动（帧 `FE 55 10 [字节] 55 FE`，显示在眼屏），与方向/电机同一条链路。接入厂家表里 18 个已确认表情（生气/笑一个/比个心/眨眼/开心/难过/再眨眼/装酷/坏笑/放烟花/石头剪刀布/害怕/晕头/音乐/害羞/情人节/玫瑰/吃零食）。
- **控制台「表情」区**：18 个表情一键触发，带「已发送」提示。
- **表情探索器**：输入任意 1 字节十六进制或用「上/下一字节」逐个扫，用于实车标定「好热/好冷」等未公开表情字节（对照小程序「特殊指令」入口）。
- **场景助手新增 17 项表情动作**：自定义规则可直接选「表情·开心」等（0x13 音乐仅动作不计）。「空调互动」改用表情——调高→点头+开心、调低→哆嗦+装酷（去掉原氛围灯灯色）。
- **隐藏氛围灯**：主页设备列表的「氛围灯 · ET-ROBOT-01」行与测试台「底灯色」色块整块隐藏，聚焦机器人本体；控制台氛围灯完整控制保留。

## v5.7 (versionCode 30) — 2026-06-22
- **主页一键绑定**：已连接的设备旁直接给「绑定此设备」按钮，连上即可绑定，免去再进扫描页指派；已是绑定设备时按钮显示「已绑定」并禁用。
- **设备改名更直观**：「运动板」全程改称「机器人本体」（主页/控制台/绑定页/清除按钮一致），氛围灯保持灯效控制语义；主页每台设备加副标题（身体动作 · MIDBOW1S / 灯效控制 · ET-ROBOT-01）。
- **控制台开关化 + 发送反馈**：本质是开关的项（电机供电/声音/唤醒音/自适应律动）由「开/关」双按钮改为单个开关，更简洁；所有按钮/开关/滑杆操作加「已发送」提示——蓝牙单向下发无状态回读，提示仅表示命令已发出。「自适应律动」默认关（避免莫名律动）。
- **二级页加返回键**：控制台/绑定/场景助手/规则编辑页统一加「← 返回」。
- **新增「空调互动」预置规则**：监听空调设定温度，调高→暖色点头、调低→冷色哆嗦，增加与车机的互动感（升级后自动出现在场景助手，可单独开关/删除）。

## v5.6 (versionCode 29) — 2026-06-22
- **语音联动改走免授权通道（根因修复）**：v5.5 把车机语音 SDK 打包进来后实测仍拿不到状态——车机统一核心服务对第三方应用有密码学授权门槛（需车机厂商私钥签发、与本应用包名+证书绑定的 license 文件才放行），平台签名/system uid 均无法绕过。本版彻底放弃该 SDK，改用两条免授权旁路：
  - **状态**：起 logcat 子进程，按语音引擎的事件主题字串识别 唤醒/聆听/说话/空闲，驱动机器人表情（需 READ_LOGS，平台签名授予）。
  - **唤醒方向**：唤醒时直接读语音守护进程的偏好文件取声源方位（主驾/副驾/后排），机器人朝说话人转向（同 system uid 可读）。
- 移除随包打包的闭源语音 SDK，APK 更小、也不再依赖厂商私有产物。
- 车型相关的包名/文件名/日志主题字串集中到一个不入库的配置类，公开仓库只附 `.example` 模板；移植到其他车型按模板填写即可。
- 注：logcat 状态主题字串以实车日志取证为准，换车型需重新核对。

## v5.5 (versionCode 28) — 2026-06-22
- **修语音联动从未生效（根因）**：此前语音助手状态一直收不到，原因是所需的车机语音 SDK 不在车机公共 classpath 上，第三方 app 反射时类加载必然失败、从初始化就静默降级。本版把该 SDK 随 APK 打包进来，反射调用才能命中真实服务。
- **注册参数修正**：注册回调时的语义域参数由占位值改为车机样本实测有效值，避免注册被判无效而静默失败。
- 完善唤醒/聆听/思考/说话/空闲各状态的运行日志，便于实车验证是否正确驱动机器人。
- 注：所打包的语音 SDK 为车机厂商闭源产物，不随仓库分发（见 .gitignore），移植到其他车型需自备。

## v5.4 (versionCode 27) — 2026-06-22
- **设备绑定（开源支持）**：新增绑定页，扫描周围全部 BLE 设备，手动指派哪台是运动板、哪台是氛围灯并按 MAC 记忆；未绑定时回退到出厂广播名自动识别。让不同名字/MAC 的同款机器人也能开箱接入。
- **设备控制台（脱离厂商小程序）**：新增控制台，照搬小程序的直接控制能力——
  - 运动板：方向、电机通断电、声音/唤醒音开关、律动加减速、自适应律动、工作模式、特殊与快捷指令、随机动作池。
  - 氛围灯：预设色、亮度、自动亮度、灵敏度多项、校准、重启、复位、激活、同步时间、状态查询。
  - 出于安全不含 OTA 文件传输（刷砖风险）。
- **盲区接真实信号**：改用车辆实时盲区信号驱动「朝来车侧探头」，替换原先无效的设置项监听。
- **加速联动修复**：纵向加速度改为后台轮询读取，解决「急加速/急刹不触发」。
- **system 共享 uid**：声明 `sharedUserId=android.uid.system`，配合平台签名获得所需系统权限（改 uid 后旧版需先卸载再装）。

## v5.3 (versionCode 26) — 2026-06-22
- **修「没放音乐却莫名律动」**：原先把导航播报、系统提示音等都误判成「在放音乐」。改为按媒体会话与音频用途精确判定，只有真在放歌才律动，并加停止去抖。

## v5.1 (versionCode 24) — 2026-06-22
